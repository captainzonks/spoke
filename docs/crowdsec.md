# CrowdSec Integration

CrowdSec is an open-source security engine that combines behavioral analysis with
a crowdsourced threat intelligence network. It detects and blocks malicious
traffic before it reaches your services.

---

## What CrowdSec Does

**Behavioral Analysis**: CrowdSec reads your log files (Traefik access logs, system
logs) and applies detection scenarios to identify attack patterns — brute force,
path traversal, credential stuffing, port scanning, and more.

**AppSec WAF**: The AppSec component intercepts HTTP requests in-line via Traefik
and applies virtual patching rules (146+ rules covering OWASP Top 10 and known CVEs)
before requests reach your applications.

**Crowdsourced Threat Intel**: When CrowdSec identifies an attacker, it shares the IP
with the CrowdSec Central API (CAPI). In return, you receive a continuously updated
blocklist of IPs reported by thousands of other CrowdSec deployments worldwide.

**Traefik Bouncer Plugin**: The Traefik bouncer plugin checks every incoming request
against the local CrowdSec LAPI decision cache. Blocked IPs receive a 403 before
any application code runs.

---

## How Spoke Integrates CrowdSec

```
Browser → Cloudflare → Traefik (crowdsec plugin) → CrowdSec LAPI → AppSec WAF → Service
                              ↑                         ↑
                    bouncer checks IP cache      analyzes request body
```

**Services involved**:
- `crowdsec` container: runs the security engine, reads logs, manages decisions
- `crowdsec-bouncer-traefik-plugin`: Traefik plugin that checks each request
- `middlewares.yml`: defines two bouncer middlewares (`crowdsec`, `crowdsec-upload`)

**Middleware chains**: All `chain-*` middlewares in `hub/traefik/rules/middlewares.yml`
include the `crowdsec` middleware. The `chain-upload` chain uses `crowdsec-upload`
(AppSec disabled for large file uploads to avoid body size limits).

**Startup ordering**: When `CROWDSEC_ENABLED=true`, the `docker-compose.crowdsec.yml`
overlay adds `depends_on: crowdsec` to Traefik. The custom Traefik entrypoint also
waits for the CrowdSec LAPI health endpoint before starting.

---

## Default State: Disabled

CrowdSec is **opt-in**. By default (`CROWDSEC_ENABLED=false`):
- The `crowdsec` container has `profiles: [crowdsec]` and does not start
- `deploy_hub_rules.sh` deploys `middlewares_nocrowdsec.yml` (chains without CrowdSec)
- Traefik skips the CrowdSec readiness check on startup
- All middleware chains function normally, just without the bouncer step

This makes local development and testing frictionless — no CrowdSec account required.

## Prerequisites

1. **CrowdSec account**: [app.crowdsec.net](https://app.crowdsec.net) (free)
2. **LAPI key**: Generated during CrowdSec first run (stored in `secrets/crowdsec/crowdsec_lapi_key`)
3. **Online API credentials**: Retrieved from the CrowdSec console after enrolling your instance

---

## Enabling CrowdSec

### Step 1: Create a CrowdSec account

Register at [app.crowdsec.net](https://app.crowdsec.net). This gives you access to
the Central API for threat intelligence sharing.

### Step 2: Set `CROWDSEC_ENABLED=true`

In `shared/env/hub.env`:

```bash
CROWDSEC_ENABLED=true
```

### Step 3: Populate secrets

```bash
# The LAPI key is generated on first run — leave placeholder for now
# It will be auto-generated by the CrowdSec container

# Online API credentials from app.crowdsec.net → Instances → Enroll
echo -n 'your-capi-machine-id' > secrets/crowdsec/crowdsec_online_api_login
echo -n 'your-capi-password' > secrets/crowdsec/crowdsec_online_api_password

# The LAPI key is populated after first boot — see Step 5
```

### Step 4: Set the CrowdSec database password

CrowdSec does not support Docker secrets for its PostgreSQL connection. Set a
password directly in `shared/env/hub.env`:

```bash
CROWDSEC_PSQL_PASSWORD=your-strong-password-here
```

### Step 5: Build and start

```bash
make hub-rebuild CROWDSEC_ENABLED=true
```

On first start, CrowdSec generates a LAPI key. Retrieve it from the container:

```bash
docker exec crowdsec cscli bouncers list
# Or look in appdata/crowdsec/config/local_api_credentials.yaml
```

Copy the generated key to `secrets/crowdsec/crowdsec_lapi_key`, then recreate:

```bash
echo -n 'generated-lapi-key' > secrets/crowdsec/crowdsec_lapi_key
make hub-recreate CROWDSEC_ENABLED=true
```

### Step 6: Enroll your instance

```bash
docker exec crowdsec cscli console enroll <enrollment-key-from-app.crowdsec.net>
```

## Collections

Collections are pre-packaged sets of parsers, scenarios, and postoverflows.
The default Spoke collections (set in `hub.env`):

| Collection | Protects Against |
|------------|-----------------|
| `crowdsecurity/traefik` | Traefik log parsing and HTTP attacks |
| `crowdsecurity/whitelist-good-actors` | Whitelists known good crawlers |
| `crowdsecurity/linux` | SSH brute force, system log attacks |
| `crowdsecurity/appsec-virtual-patching` | 146+ OWASP/CVE virtual patches |

To add collections:

```bash
docker exec crowdsec cscli collections install crowdsecurity/nginx
# Then update CROWDSEC_COLLECTIONS in hub.env to persist across rebuilds
```

## Monitoring CrowdSec

Check active decisions (currently blocked IPs):

```bash
docker exec crowdsec cscli decisions list
```

View recent alerts:

```bash
docker exec crowdsec cscli alerts list --limit 20
```

Check registered bouncers:

```bash
docker exec crowdsec cscli bouncers list
```

View metrics:

```bash
docker exec crowdsec cscli metrics
```

If the monitoring module is deployed, CrowdSec metrics are scraped by Prometheus
and visualized in the CrowdSec Grafana dashboard.

---

## Disabling CrowdSec

To disable CrowdSec (e.g., for local testing):

```bash
# In shared/env/hub.env:
CROWDSEC_ENABLED=false

# Redeploy:
make hub-recreate CROWDSEC_ENABLED=false
```

The `crowdsec` container stops, and Traefik switches to the no-CrowdSec middleware
chains automatically.

## Troubleshooting

**Traefik fails to start with "plugin loading failures"**:
Ensure `CROWDSEC_ENABLED` matches whether the crowdsec container is actually running.
The Traefik entrypoint waits up to 120s for CrowdSec LAPI.

**403 errors on legitimate requests**:
Check if your IP is in the CrowdSec decision list:
```bash
docker exec crowdsec cscli decisions list | grep <your-ip>
# Remove a false positive:
docker exec crowdsec cscli decisions delete --ip <your-ip>
```

**AppSec blocking uploads**:
The `chain-upload` middleware uses `crowdsec-upload` which has AppSec disabled.
Use `chain-upload` for routes that accept large file uploads.
